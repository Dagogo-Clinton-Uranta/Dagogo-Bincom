Online Journal

An Authentication Project.

Design Insights

This project served as practice, to improve my ability to handle login requests using Node.js and JSON web tokens(JWT) and as a result is not UI intensive. It is subject to updates however...

  • The front-end consists of a simple centered form element for login and another form element upon login, to submit new journal entries into the database.


  • It also consisted of a page for viewing and deleting posts, which had to be authorized before viewing.


  • For the back-end, passwords were stored in the database as hashed strings (to improve app security), courtesy of the Bcrypt package.


Go to Website

Workflow

  • I wrote code for the server and three endpoints, one for registering, one for logging in and one to see posts from the site. I pushed this code up to my repository on github onto my master branch.Next, I made use of the POSTMAN tool to call my API endpoints. I setup a new database cluster on MongoDB atlas, to store and update information from my endpoints.


  • After connecting the database to my back-end code, I created a schema for my data using Mongoose package, making it accept name, email, password and data information.


  • The next phase involved parsing the register endpoint for name, email and password information, and catching errors in this process. I did this by using a try-catch block which would send back the appropriate error codes.


  • Next I made use of the Joi package for validation, using the Joi library's validate method in my already existing schema. I tested the new validation conditions using POSTMAN. I made a second git commit to master to overwrite my previous one.


  • Bcrypt js was the next package I used, to hash user passwords, as opposed to storing them directly on the database. Bcrypt was setup improperly and so I had to perform fetch and merge operations from the master branch on my Git repository to continue from the last working version of the code. I used an async function and await expression on the bcrypt method to pass tests successfully.


  • Lastly, JSON web tokens were introduced to the project to issue users tokens upon login. The token was made to expire within an hour. token validation was implemented as a middleware on the posts route, and the final commit was made to the master branch.
